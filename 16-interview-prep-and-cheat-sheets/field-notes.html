Module 01

Fundamentals & Architecture

driver / executor
Driver plans & schedules; executors run tasks against partitions. In local[*] both collapse into one JVM, executors become threads.
transform vs action
Transformations (.filter, .select) are lazy, just recorded. Actions (.count, .collect) trigger real execution.
stage boundary
A shuffle — any wide transform (groupBy, join, orderBy, repartition).
why DataFrame > RDD
Schema + declarative ops let Catalyst inspect and rewrite the whole plan; RDD closures are opaque to the optimizer.
.explain()
PushedFilters = Catalyst moved your filter into the scan itself.
Module 02

Reading, Writing & Schemas

inferSchema
Costs a full extra pass over the data — and a ZIP code "00501" silently becomes int 501. No warning.
CSV vs Parquet
Parquet is columnar — can skip unread columns entirely. CSV must parse whole rows regardless.
multi-line JSON
Needs .option("multiLine","true") or every line fails to parse.
corrupt-record .count()
Unreliable, verified — even after .filter(). Force .cache() before counting.
schema drift, no merge
Silently keeps ONE file's schema for the whole read — drops a real column from every row.
coalesce vs repartition
Coalesce avoids a full shuffle but can't fix already-uneven partitions.
Module 03

Core Transformations

withColumn
Second arg must be a Column — wrap constants in lit(...).
& | ~
Binds tighter than comparisons — every condition needs its own parens, no exceptions.
compare vs NULL
Evaluates to NULL, not False — silently excluded, falls through when() chains to .otherwise().
.union()
Matches columns by position, not name — silent mislabeling. Use unionByName().
temp view scope
Scoped to the creating SparkSession only.
Module 04

Spark SQL

injection-safe SQL
Use args={"x": val} named params, never f-strings against untrusted input.
UNION vs ALL
Bare SQL UNION dedups by default (unlike DataFrame .union()). Default to UNION ALL.
NOT IN + NULL
One NULL in the subquery collapses the result to zero rows for everyone. Use NOT EXISTS.
CAST('x' AS DOUBLE)
Returns NULL, no error — indistinguishable from genuinely missing.
DROP TABLE
Deletes files for MANAGED tables; leaves them for EXTERNAL. Check DESCRIBE EXTENDED first.
Module 05

Joins Deep Dive

LEFT SEMI vs INNER
Semi = one row per matching left row (membership check). Inner = one row per match (row-multiplying).
ambiguous column
on="col" auto-coalesces; a join condition keeps both sides — alias both, reference via alias.
autoBroadcastJoinThreshold
Default 10MB — auto-broadcasts with zero hint. Stale size estimates risk executor OOM.
skew symptom
One task stays running long after every sibling in the same stage finishes.
NULL == NULL join key
Evaluates NULL → never matches, even two literal nulls against each other.
missing join condition
crossJoin.enabled defaults true — silently runs as a full cartesian product.
Module 06

Partitioning & Shuffling

input partitions
Governed by maxPartitionBytes (128MiB default) — file size ÷ threshold.
repartition(n, col)
Hash-partitions — same key value always lands in the same output partition.
shuffle.partitions
Defaults to 200, fixed, unrelated to data size. Verified 200 partitions for 4 groups on 15 rows.
AQE coalescing
Shuffle still WRITES the configured count — AQE only merges small partitions back down on read.
skew detection
.rdd.glom().map(len) — exact per-partition row counts, not inferred from timing.
salting
Needs a two-stage aggregation (by salt, then by real key) to recover the correct total.
Module 07

Window Functions

rank() vs dense_rank()
Rank skips ahead by the tie count; dense_rank always +1. Verified on a real 3-way tie.
row_number() ties
Non-deterministic assignment without a unique tiebreaker in orderBy.
default frame
RANGE, not ROWS — tied rows get the SAME cumulative total. Silent, data-dependent bug.
safe running total
Always specify .rowsBetween(unboundedPreceding, currentRow) explicitly.
no partitionBy
Exchange SinglePartition — every row funnels into one partition.
top-N per group
Use row_number(), not rank() — rank can return MORE than N on a boundary tie.
Module 08

UDFs & Pandas UDFs

@udf(returnType=)
Never checked against the real return value — wrong type silently becomes NULL everywhere.
.explain() signature
Row UDF → BatchEvalPython. Pandas UDF → ArrowEvalPython.
Pandas UDF speedup
Verified modest, ~5–15% for numeric work locally — not the "10–100x" folklore figure.
NaN from a pandas UDF
Lands as genuine Spark NULL — check is None, not a NaN comparison.
Module 09

Performance Tuning

.explain() timing
Before an action under AQE: isFinalPlan=false. After: =true + AQEShuffleRead coalesced.
.cache() default level
MEMORY_AND_DISK_DESER, NOT MEMORY_ONLY — that's the RDD API's default.
spark.conf.get(parallelism)
Throws — it's a core property. Use sparkContext.defaultParallelism.
maxPartitionBytes
Controls INPUT partitions — a distinct knob from shuffle partitions.
Module 10

Structured Streaming

streaming actions
.count()/.show() raise AnalysisException — must writeStream.start().
Trigger.ProcessingTime
First batch fires immediately regardless of interval; only later batches wait the full interval.
Trigger.AvailableNow
Stops ITSELF once the backlog drains — isActive → False, no .stop() needed.
watermark + late data
Within tolerance → merges into the open window. Past it → dropped silently, but still counted in numInputRows.
foreachBatch
Multiple uncached actions on batch_df double real work AND the reported metric.
Module 11

Delta Lake / Lakehouse

what a Delta table IS
Parquet files + _delta_log/ — the log is what enables ACID, versioning, MERGE.
append + mergeSchema
On an existing key: creates a DUPLICATE row, not an update. MERGE is the real upsert.
OPTIMIZE vs VACUUM
Optimize compacts but keeps old files (16→17). Vacuum deletes them (17→1) — has a retention safety check.
concurrent MERGE
Reproduced live: one thread wins, the other gets ConcurrentAppendException.
RESTORE
Adds a NEW version — doesn't erase the one it restores from.
Module 12

Data Engineering Patterns

idempotent loads
Naive append run twice DOUBLES rows (2→4). replaceWhere stays correct (2→2).
SCD2 trap
Changed-keys DataFrame reused after a MERGE finds nothing — even .cache() doesn't fix it. .collect() first.
quality gate design
Must RAISE before write, and check row-count volume — not just per-row validity.
dead-letter split
Broken JSON syntax nulls the whole row; a wrong-typed field nulls only that field. _corrupt_record catches both.
Module 13

Testing PySpark Code

fixture scope
session-scoped amortizes the ~2.8s JVM startup once instead of per test.
assert_df_equality
Fails by default on row order, column order, AND nullable flags — even identical data.
float columns
0.1 + 0.2 != 0.3 exactly. Use assert_approx_df_equality(precision=...).
empty-batch trap
A per-row-only quality gate trivially PASSES an empty DataFrame — add a volume check.
Module 14

Production & Deployment

--conf / --py-files
Both verified to take real effect — configs readable inside the job, modules genuinely importable.
cluster sizing
~5 cores/executor is the standard sweet spot — balances GC pauses against per-JVM overhead.
retry safety
Airflow retries is only safe if the job is already idempotent — otherwise it automates the bug.
deploy mode
cluster mode survives the submitting machine disconnecting; client mode does not.
